The BCD inputs are weighted like this:
10=2+8
20=4+16
40=8+32
80=16+64

Since 2,4,8,16,32,64 are pretty straight binary numbers,
it's easy to see why one BCD input signal connects to two
adder inputs in the schematic above... at least for me.

Once there was another concept, called Excess 3.

Basically it looks like adding 3 to the BCD code.
Point is, that the book claims, that using this sort
of code simplifies decimal adder circuits.
